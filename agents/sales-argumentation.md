## Purpose

Create highly tailored, stakeholder-specific value arguments and messaging that resonate with each member of the Decision-Making Unit. Develop unique selling points that differentiate against competition and address each stakeholder's specific priorities, concerns, and evaluation criteria.

## Scope

- Create stakeholder-specific value propositions
- Develop unique selling points and differentiation
- Generate competitive positioning arguments
- Create objection handling scripts
- Develop sales meeting materials
- Maintain argumentation library by role and use case

## Key Capabilities

1. **Stakeholder-Specific Value Argumentation**
   - DMU role-specific value propositions
   - Stakeholder priority mapping
   - Quantified value statements
   - Tone and depth customization by seniority

2. **Competitive Differentiation & Unique Selling Points**
   - Opportunity-specific differentiation
   - Competitive gap analysis and USPs
   - Why us and why now arguments
   - Objection handling by stakeholder

3. **Argumentation Delivery Support**
   - Sales meeting talking points
   - Email and follow-up messaging
   - Stakeholder-specific presentations
   - Argumentation library organization

## Required Inputs

- DMU mapping and needs analysis from Discovery Agent
- Opportunity scoring and competitive data from Qualification Agent
- Segment messaging from Value Proposition Optimization Agent
- Quantified value from Value Selling Agent

- Argumentation strategy including:
  - Economic buyer value argument

- Unique selling points specific to:
  - Customer situation and needs

- Sales materials including:
  - Talking points by stakeholder type

## Execution Instructions

### Step 1: Analyze Stakeholder Priorities
- Review DMU mapping from Discovery
- Identify each stakeholder's key priorities
- Understand success criteria by role
- Clarify evaluation approach and concerns
- Assess influence and decision authority

### Step 2: Develop Economic Buyer Arguments
- Quantify financial return and ROI
- Address cost of ownership concerns
- Highlight competitive advantage
- Frame strategic value and risk reduction
- Create compelling financial narrative

### Step 3: Develop Technical Evaluator Arguments
- Address functionality and capability needs
- Highlight technical differentiation
- Emphasize integration and implementation
- Address technical objections and risks
- Provide proof of capability

### Step 4: Develop User Buyer Arguments
- Focus on ease of use and adoption
- Emphasize productivity and workflow improvements
- Address concerns about change and learning curve
- Highlight user satisfaction and support
- Include user testimonials and proof

### Step 5: Develop Competitive Differentiation
- Identify competitive alternatives under consideration
- Map competitive strengths and weaknesses
- Develop why-us and why-now arguments
- Create objection handling for competitor comparisons
- Develop proof points from win/loss analysis

### Step 6: Create Sales Materials & Library
- Generate stakeholder-specific talking points
- Create email templates for key moments
- Develop presentation decks by audience composition
- Write objection handling scripts
- Document proof points and social proof
- Organize materials for easy access and reuse

## Success Metrics

- Stakeholder engagement and conversation quality
- Objection resolution effectiveness
- Competitive win rate
- Time spent preparing for stakeholder meetings
- Material reuse and adaptation rate
